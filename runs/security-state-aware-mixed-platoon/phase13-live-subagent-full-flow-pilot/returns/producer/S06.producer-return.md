The current object granularity is coherent: reader-facing concepts move from scene/object view, to equations and algorithm, to validation interpretation. The effective explanation ladder is: physical mixed platoon → controller-visible sources → authority weights → fused predecessor estimate → MPC update → aggregate validation and boundary claims. This reduces flat repetition because “fresh V2X,” “memory,” “local sensing,” and “dynamics prior” do not merely recur as labels; they take different functions in each section. ## Limitations
The main remaining risk is cognitive load around object naming. The manuscript uses closely related objects: source channels, authority weights, fused estimates, memory state, quality loss, and validation metrics. These are technically separated, but a reader may still blur online controller objects with offline scoring objects unless each section preserves its assigned budget. Section 03 already states that aggregate score is offline and not a controller input (`manuscript/sections/03_problem_formulation.tex:23-23`); S06 should keep that distinction visible. Section 06 also needs continued restraint: the mechanism ladder is explicitly not a full component-ablation proof (`manuscript/sections/06_results_discussion.tex:78-78`).

## Downstream usefulness
This S06 candidate is useful for downstream writing units because it gives a concrete allocation rule: Section 03 should carry definitions and object boundaries; Section 04 should carry mechanism, equations, and algorithmic granularity; Section 06 should carry evidence granularity, comparator interpretation, and claim limits. The load budget should prevent over-explaining all four authority sources every time they appear.
