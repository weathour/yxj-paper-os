# yxj-paper-os Architecture Philosophy

The product is a thin schema-0.3 cognitive compiler over one public skill and six
public Markdown files. Its goal is a detailed, traceable design package that can
control downstream writing at paragraph, important-sentence-frame, language, visual,
and cross-surface level while stopping before paste-ready manuscript prose.

## Repository / template / owner triad

### Repository authority

Current project artifacts determine what exists, what can be claimed, the scientific
referent, uncertainty and adverse findings, and the wording ceiling. Scientific
material and claims use `M-*` and `C-*` records. Planning never fills evidence gaps
with template literature or rhetoric.

### Template authority

Eligible official guidance and exemplar literature determine how supported content
may be organized and presented. The primary path is model-semantic full-text reading
with provenance, locators, access state, fingerprints, freshness, limitations, and a
compact public projection. All `TPL-*`, `TOBS-*`, semantic `TRULE-*`, quantitative
outputs, and generic fallback remain `design_only`.

The deterministic analyzer is a conditional instrument, not the product center. Use
it only for an explicit measurable design question and preserve metric, partition,
denominator, missingness, comparability, and analysis-mode ceilings. Analyzer absence
is neutral for semantic-primary, generic-fallback, and grounded-not-applicable scopes.

### Owner authority

Human-in-the-loop control protects consequential choices without turning every design
step into a questionnaire. Ask one focused question only for
`scientific_commitment`, `argument_spine`, `material_local_tradeoff`, or
`deliberate_divergence`. Routine grounded reversible discovery, adaptation,
terminology repair, important-paragraph selection, accessibility completion, and
optional-analyzer use remain agent-resolved with origin preserved.

## Detailed-design stopping boundary

Every planned paragraph receives a functional contract; adaptively important
paragraphs receive controlled frames containing slots, relations, payload IDs,
hedges, and forbidden realizations. Applicable language, visual/caption/accessibility,
traceability, template-rule, and soft-budget surfaces are completed or explicitly
not applicable. The compiler stops there. It does not produce polished sentences,
full paragraphs, captions, titles, abstracts, or table prose.

## Stable invariants

- exactly one public skill and six public workspace files;
- stable D00-D19 identities, index columns, and public status vocabularies;
- 00 is sole readiness authority, 02 sole scientific ceiling, 03 detailed authority,
  and 04 pointer/count/blocker only;
- origin, support, resolution, grounding, access, and freshness remain distinct;
- five template modes and seven detailed surfaces are scope-local;
- dependency changes stale only declared linked scopes/records;
- schema 0.2 is preserved and non-destructively recompiled, never auto-migrated;
- validation reports declared structural consistency, not semantic completeness.

No fixed scan order, exposed stage trace, global template gate, universal critical
set, mandatory subagent sequence, daemon/watcher/runtime graph, manuscript drafting,
external-skill execution, or quality/venue/acceptance score.
